# Security Policy

PakScape treats archive metadata, paths, and payloads as untrusted input. Security fixes are supported on the latest published release and the current `main` branch.

## Reporting a vulnerability

Please use the repository's **Security** tab to submit a private vulnerability report. Include the affected PakScape version and platform, the smallest archive that reproduces the problem, the observed impact, and any relevant crash log.

If private vulnerability reporting is unavailable, open a minimal issue asking the maintainers for a private contact channel. Do not publish exploit details or a malicious archive in a public issue.

You should receive an acknowledgement before any disclosure timeline is agreed. Please allow the maintainers time to reproduce, fix, and distribute an update before publishing details.
